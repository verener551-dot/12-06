# Домашнее задание к занятию "`Репликация и масштабирование. Часть 1`" - `Шалыгин Денис`

### Задание 1

На лекции рассматривались режимы репликации master-slave, master-master, опишите их различия.

Ответить в свободной форме.
### Ответ

1 Архитектура 
Master-Slave - один сервер выполняет роль источника данных остальные — его реплики.
Master-Master - каждый одновременно является и источником и приёмником данных.

2 Направление репликации

Master-Slave - Односторонняя синхронизация. Данные передаются только от master к slave.
Master-Master - Многосторонняя синхронизация. Изменения с каждого master реплицируются на все остальные узлы.

3 Масштабируемость

Master-Slave - Эффективно масштабирует операции чтения. Добавление slave-серверов распределяет нагрузку запросов.
Master-Master - Теоретически масштабирует операции записи, но на практике ограничена накладными расходами на разрешение конфликтов и синхронизацию.

4 Отказоустойчивость
Master-Slave - При выходе из строя master-сервера система теряет возможность записи до момента ручного или автоматического переключения одного из slave.
Master-Master - При отказе одного из master-серверов система сохраняет полную функциональность на оставшихся узлах.

5 Задержка репликации
Master-Slave - Задержка актуальна только для slave-серверов. Время между записью на master и её отражением на slave.

Master-Master - Задержка может влиять на работу всех узлов и усугублять конфликтные ситуации.
